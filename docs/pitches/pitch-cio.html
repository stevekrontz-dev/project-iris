Executive Summary

KSU Research is an on-premise research collaboration platform with AI-powered matching. Unlike cloud SaaS solutions, all data remains within KSU infrastructure. The AI engine (IRIS) runs locally using open-source models - no data is sent to OpenAI, Google, or any external service.

Security Considerations

Data Residency: All data stays within KSU infrastructure. Zero external API calls for AI inference. No cloud dependencies.

Authentication & Authorization

  • SSO integration via SAML 2.0 (Shibboleth compatible)
  • Role-based access control (Researcher, Admin, Institution Admin)
  • Row-level security in PostgreSQL
  • Session management with secure, httpOnly cookies

Data Protection

  • TLS 1.3 for all connections
  • Database encryption at rest (if required)
  • No PII stored in logs
  • Full audit trail of data access
  • User-controlled privacy settings

Compliance

  • FERPA: No student data collected; faculty control their data
  • Export Control: No data leaves US jurisdiction
  • IRB: Platform itself doesn't conduct research; just facilitates connections
  • SOC 2: Architecture supports certification if needed

Questions I Anticipate

Q: Can we audit the AI models?
A: Yes. We use open-source models (Llama 3.2, Nomic Embed). Full transparency into how recommendations are generated.

Q: What about updates and patches?
A: We provide versioned releases. You control when to update. No automatic cloud updates.

Q: What if Ollama/Llama changes licensing?
A: The models are already downloaded locally. Future licensing changes don't affect deployed instances. We also support alternative models.
